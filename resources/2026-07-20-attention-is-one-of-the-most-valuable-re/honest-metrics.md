# Honest Attention Scorecard

A lightweight framework for teams that want to stop optimizing for extraction and start optimizing for value delivered. Copy this into a doc or issue and fill it in per feature.

## The Core Reframe

Extractive metrics reward keeping people *in*. Honest metrics reward helping people *succeed and leave satisfied* — which is what actually earns durable return.

## Metric Pairs (track the healthier partner)

| Extractive metric | Honest counterpart | Why the counterpart is better |
|-------------------|--------------------|-------------------------------|
| Time on site | Time-to-goal (lower is better) | Rewards helping users finish, not linger |
| Sessions per day | Task success rate | Frequency without success is friction, not value |
| Notification open rate | Notifications the user *kept enabled* | Trust survives when signal is high |
| Daily active users | Weekly retained-for-a-reason users | Filters out compulsive re-opens |
| Feed scroll depth | Voluntary return rate after a clean exit | Return by choice, not by loop |

## Pre-Ship Checklist

- [ ] Does this feature have a clear, satisfying stopping point?
- [ ] Is every urgency claim (countdowns, scarcity) factually true?
- [ ] Can a user leave / cancel / export as easily as they joined?
- [ ] Would this feature survive if we removed all notifications?
- [ ] Are we measuring whether the user accomplished their goal?

## A Simple Health Ratio

```
Honest Engagement Ratio = (Users who completed a goal) / (Total sessions)
```

A rising ratio means people succeed more per visit. A falling ratio (even with rising "engagement") is a warning that you're generating activity, not value.

## Team Ritual

Once a quarter, ask in a retro:
> "Which of our wins this quarter were users returning because they chose to, and which were them returning because we made leaving costly?"

Write the answer down. The gap between those two is your extraction debt.
